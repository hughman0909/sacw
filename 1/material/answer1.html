問題 1

cat.png / mystery.zip / kindai.heic は実際に何のファイル？

ディレクトリ1にある3つのファイルについて、実際のファイル形式をそれぞれ答えてください。

ヒント1

file コマンドはファイルの先頭バイト（マジックバイト）を読んでファイル形式を判定します。拡張子は一切見ません。

ヒント2

ディレクトリ1で次を実行してみましょう：
file cat.png mystery.zip kindai.heic

ヒント3

出力に JPEG image data / PNG image data / PDF document / GIF image data のいずれかが表示されます。

答えを見る
  • cat.png → JPEG（先頭バイト: FF D8 FF）
  • mystery.zip → PDF（先頭バイト: 25 50 44 46 = %PDF-）
  • kindai.heic → GIF（先頭バイト: 47 49 46 38 = GIF8）

拡張子はOSへの「ヒント」に過ぎず、中身とは無関係。悪意あるファイルはよくこれを偽造します。

問題 2

sample1 と sample2、sample1 と sample3 — 同じファイル？異なるファイル？

それぞれのペアについて「同じ」か「異なる」かを選び、理由も考えてみてください。

ヒント1

ファイルの同一性はハッシュ値で比較できます。shasum コマンドを使いましょう。

ヒント2

shasum -a 256 sample1 sample2 sample3 を実行して3つのハッシュ値を比べてみましょう。

ヒント3

ハッシュが完全一致 → 同一ファイル。1バイトでも違えばハッシュは全く異なります（雪崩効果）。cmp sample1 sample3 でも差が確認できます。

答えを見る

sample1 と sample2：同じファイル
shasum -a 256 sample1 sample2 のハッシュ値が完全一致する。


sample1 と sample3：異なるファイル
shasum -a 256 sample1 sample3 のハッシュ値が全く異なる。内容は1バイトだけ違うが、SHA-256の雪崩効果で全く別の値になる。

「ファイルサイズが同じ＝同じファイル」は誤り。マルウェア解析・フォレンジクスでは SHA-256 比較がデファクトスタンダード。
