問題 8

normal と packed_normal に strings を実行して違いを確認せよ

ディレクトリ4にある normal と packed_normal に対して strings を実行し、出力の違いを観察してください。

ヒント1

strings normal | head -30 を実行してみましょう。コードに書かれた文字列が読めるはずです。

ヒント2

次に strings packed_normal | head -30 を実行して比較しましょう。先頭に UPX! などの文字列が見えます。

観察した違い：
答えを見る

normal（パックなし）

  • This is a perfectly normal
  • program.
  • [INFO] Process completed
  • [WARN] Something went wrong.
  • [ERROR] Fatal error occurred.
  • panicked at
  • RUST_BACKTRACE
  • ... Rust ランタイム文字列 ...

packed_normal（UPXパック済み）

  • 3UPX!
  • $Info: This file is packed
  • with the UPX executable
  • packer
  • （元の文字列は見えない）
  • （圧縮されたデータの断片）
  • ...

パッキング済みでは元のコードが圧縮されており、アプリ固有の文字列が見えなくなる。代わりにUPX関連の文字列（UPX!、$Info:）のみが見える。

これがマルウェアがパッキングを使う理由。strings・xxdによるシグネチャ検知を回避できる。

問題 9

strange バイナリはどんなことをするファイルか説明せよ

ディレクトリ4にある strange バイナリが何をするプログラムかを調べてください。

ヒント1

strings strange を実行してみましょう。パックされているので有用な文字列はほとんど見えません。

ヒント2

元ファイルはコピーしてからアンパックしましょう：

cp strange strange_copy
upx -d strange_copy
ヒント3

アンパック後に文字列を確認：

strings strange_copy | grep -E "運勢|吉|おみくじ|ようこそ"

日本語文字列が複数見えるはずです。

どんなプログラムか：
答えを見る

今日の運勢を表示するおみくじプログラム

アンパック後の strings で判明する文字列：
cp strange strange_copy
upx -d strange_copy
strings strange_copy | grep -E "運勢|吉|おみくじ|ようこそ"
# → ようこそ、今日のおみくじへ！
# → あなたの今日の運勢は...
# → 大吉 / 中吉 / 小吉 / 吉 / 末吉 / 凶
# → 良い一日を！ さようなら～

「アンパック → strings」は表層解析の基本コンボ。実際のマルウェア解析でもまったく同じ流れ。
