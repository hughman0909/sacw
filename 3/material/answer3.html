問題 5

PDFのファイルシグネチャはなんですか？

PDFファイルの先頭に必ず存在するバイト列（ファイルシグネチャ）を調べてください。ASCII表示と16進数の両方を答えてください。

ヒント1

ファイルシグネチャはファイルの先頭に固定的に置かれる特定のバイト列です。file コマンドはこれを見てファイル形式を判断します。

ヒント2

1章の mystery.zip は実はPDFです。xxd で先頭を確認しましょう：

xxd ../1/mystery.zip | head -1
ASCII表示：
16進数（先頭5バイト）：
答えを見る
ASCII16進数（先頭5バイト）
%PDF-25 50 44 46 2D

通常は %PDF-1.3 や %PDF-1.4 のようにバージョン番号が続きます。

問題 6

dog.pdf を修復してください

ディレクトリ3の dog.pdf は壊れていて開けません。修復してファイルを開き、一番上に表示されている太文字のテキストを答えてください。

ヒント1

まず現状を確認しましょう：

xxd dog.pdf | head -1

正常なPDFなら先頭が 2550 4446（%PDF）のはずです。

ヒント2

現状の先頭4バイトは 58 58 58 58（XXXX）になっています。正しい値 25 50 44 46（%PDF）に書き換える必要があります。

ヒント3

コピーを作ってから修復することを推奨します：

cp dog.pdf dog_fixed.pdf
python3 -c "
with open('dog_fixed.pdf','r+b') as f:
    f.seek(0x00)
    f.write(b'\x25\x50\x44\x46')
"
file dog_fixed.pdf
一番上の太文字テキスト：
答えを見る

壊れている箇所：先頭4バイトが 58 58 58 58（XXXX）になっている

正しい値：25 50 44 46（%PDF）

方法1：Python
cp dog.pdf dog_fixed.pdf
python3 -c "
with open('dog_fixed.pdf','r+b') as f:
    f.seek(0x00)
    f.write(b'\x25\x50\x44\x46')
"
open dog_fixed.pdf  # → 一番上に「I'm a dog PDF!」と表示される
方法2：コマンドライン
( printf '\x25\x50\x44\x46'; tail -c +5 dog.pdf ) > dog_fixed.pdf
open dog_fixed.pdf  # → 一番上に「I'm a dog PDF!」と表示される

答え：I'm a dog PDF!

難 — 問題 7

problem ファイルを修復し、隠された FLAG{} を探せ

ディレクトリ3にある problem は壊れています。元々何の形式のファイルだったでしょうか？また、FLAG{} 形式で隠されている情報を探してください。

ヒント1

xxd problem | head -1 を実行してみましょう。先頭4バイトは壊れていますが、その後ろに手がかりが残っています。

ヒント2

先頭の後ろに 74 65 20 66 6F 72 6D 61 74 20 33 00（ASCII: te format 3.）が続いています。これは何かのシグネチャの後半部分です。

ファイル形式：
見つけた FLAG：
答えを見る

ファイル形式：SQLite データベース

先頭4バイトが壊れているが、続く te format 3\0 から SQLite シグネチャの後半と判断できる。

修復：
cp problem problem.db
python3 -c "
with open('problem.db','r+b') as f:
    f.seek(0)
    f.write(b'\x53\x51\x4C\x69')
"
file problem.db   # → SQLite 3.x database
sqlite3 problem.db ".tables"
# → config  employee_records  network_traffic  system_logs  users
FLAG の場所：
sqlite3 problem.db "SELECT * FROM employee_records WHERE notes LIKE '%FLAG%';"
# → 42|Tanaka Hiroshi|...|FLAG{y0u_f0und_the_secret_record}

FLAG：FLAG{y0u_f0und_the_secret_record}

employee_records テーブルの id=42、Tanaka Hiroshi の notes カラムに含まれている。
