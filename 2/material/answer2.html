第2章 — 文字列から痕跡を探そう

回答を入力して確認ボタンを押すと、答えを見ることができます。

問題 3

ディレクトリ2のバイナリファイルの中に含まれるWebサイトへのURLを探せ

ディレクトリ2に含まれるバイナリファイルの中から、あるWebサイトへのURLだと考えられる文字列を探し出してください。

ヒント1

strings コマンドはバイナリの中から印字可能なASCII文字列（4文字以上連続）を抽出します。URLも文字列として埋め込まれていれば取り出せます。

ヒント2

URLは http:// や https:// で始まります。grep で絞り込みましょう：

strings urlbin | grep http
ヒント3

ディレクトリ内の全ファイルに一括適用：

for f in *; do echo "--- $f ---"; strings "$f" | grep -E "https?://"; done
見つけたURL：
答えを見る

https://www.kindai.ac.jp/

urlbin に埋め込まれています。

strings urlbin | grep -E "https?://"
# → https://www.kindai.ac.jp/

マルウェアでは通信先のC2サーバーURLがハードコードされることがあり、strings 一発で特定できる場合がある。

問題 4

ディレクトリ2のバイナリがそれぞれ何の言語で書かれているか特定せよ

ディレクトリ2に含まれる各バイナリファイルが何のプログラミング言語で書かれているかを特定してください。

ヒント1

各言語はコンパイル時に特有の文字列を残します：

Rust → panicked at、/rustc/、RUST_BACKTRACE

Go → GOROOT、go1.、runtime.gopanic

Swift → swift が含まれる文字列

C → 上記どれも出ない

ヒント2 
strings candidate_a | grep -Ei "rust|cargo|GOROOT|go1\.|swift|panicked"

ヒットした文字列でどの言語か判断しましょう。

ヒント3 
for f in candidate_a candidate_b candidate_c candidate_d candidate_e; do
  echo "=== $f ==="
  strings "$f" | grep -Ei "rust|GOROOT|go1\.|swift|panicked" | head -3
done
candidate_a：
candidate_b：
candidate_c：
candidate_d：
candidate_e：
答えを見る
ファイル言語根拠
candidate_aC特有の文字列なし（動的リンク）
candidate_bGoGOROOT、go1.、runtime.gopanic
candidate_cRustpanicked at、RUST_BACKTRACE、/rustc/
candidate_dJavajava/lang/Object、candidate_d.java
candidate_eSwiftswift を含む文字列が見える

Rust/Go は静的リンク（ランタイムを実行ファイルに埋め込む）のため strings の出力が多い。C/Swift は動的リンクのため少ない。
