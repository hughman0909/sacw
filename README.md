# 表層解析ワークショップ

ようこそ！このワークショップでは、ファイルの **表層解析 (surface analysis)** を実際に手を動かして練習します。
バイナリを実行することなく、`file` / `strings` / `xxd` などのツールだけで「そのファイルが何者なのか・何をするのか」を読み解く力を養うのが目的です。

---

## 環境準備

受講者の環境は **macOS** を想定しています。以下のツールを事前にインストールしておいてください。

### macOS 標準で入っているもの（追加インストール不要）

| ツール | 用途 |
|---|---|
| `file` | ファイルの種類を判定する |
| `strings` | バイナリから印字可能な文字列を抽出する |
| `xxd` / | バイナリを16進ダンプして覗く |
| `shasum` | ファイルのハッシュを計算する（同一性チェック） |
| `unzip` | zipファイルを展開する |

### 追加インストールが必要なもの

```bash
brew install upx
```

## 章立て

| 章 | テーマ | 主な道具 |
|---|---|---|
| 1章 | 拡張子の嘘を見破る / ファイル同一性 | `file`, `xxd`, `shasum` |
| 2章 | バイナリからの情報抽出 | `strings` |
| 3章 | ファイルシグネチャと破損ファイルの修復 | `xxd`, バイナリエディタ |
| 4章 | パッキング (UPX) | `strings`, `upx` |
| 5章 | 総合問題：壊れたファイル群の調査 | 全部 |

## よく使うコマンド早見表

```bash
# 今いるディレクトリにある<対象ファイル>に移動する
cd <対象ファイル>

# 今いるディレクトリのファイル一覧を表示する
ls

# ファイルをデフォルトのアプリで開く
open cat.png

# コマンドの出力を次のコマンドに渡す（パイプ）
strings a.txt | grep "flag"
#         ↑の出力を grep に渡して "flag" を含む行だけ表示
```
