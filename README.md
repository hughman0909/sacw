# 表層解析ワークショップ

ようこそ！このワークショップでは、ファイルの **表層解析 (surface analysis)** を実際に手を動かして練習します。
バイナリを実行することなく、`file` / `strings` / `xxd` などのツールだけで「そのファイルが何者なのか・何をするのか」を読み解く力を養うのが目的です。

> **⚠️ 注意：このワークショップ内のバイナリは絶対に実行しないでください。**
> 全ての問題は実行せずに解けるように設計されています。
> （ファイルを開く、`file`コマンドで調べる、`strings`で文字列を抽出する、`xxd`で中身を覗く、などはOKです。）

---

## 環境準備

受講者の環境は **macOS** を想定しています。以下のツールを事前にインストールしておいてください。

### macOS 標準で入っているもの（追加インストール不要）

| ツール | 用途 |
|---|---|
| `file` | ファイルの種類を判定する |
| `strings` | バイナリから印字可能な文字列を抽出する |
| `xxd` / `hexdump` | バイナリを16進ダンプして覗く |
| `shasum` | ファイルのハッシュを計算する（同一性チェック） |
| `unzip` | zipファイルを展開する |
| `sqlite3` | SQLiteデータベースを開く |

### 追加インストールが必要なもの

```bash
brew install upx
```

### あったら便利なもの（任意）

VSCode を使う場合は、HEX エディタ拡張機能（`ms-vscode.hexeditor`）を入れておくと
バイト編集が楽になります。

## よく使うコマンド早見表

```bash
# ファイル種別を判定
file ファイル名

# 印字可能な文字列を全部取り出す
strings ファイル名

# 4文字以上の連続印字可能文字を取り出す（デフォルト動作）
strings -n 4 ファイル名

# 先頭16バイトを16進で見る
xxd ファイル名 | head -1
head -c 16 ファイル名 | xxd

# SHA-256ハッシュを計算
shasum -a 256 ファイル名

# UPXでアンパック
upx -d ファイル名

# zipの中身を見る（展開せず）
unzip -l ファイル名.zip

# SQLiteを開く
sqlite3 ファイル名
# 中で「.tables」と打つとテーブル一覧、「SELECT * FROM テーブル名;」で中身を見る
```

## ファイルシグネチャ参考表

| 形式 | 先頭バイト（16進） | 表示 |
|---|---|---|
| PNG  | `89 50 4E 47 0D 0A 1A 0A` | `.PNG....` |
| JPEG | `FF D8 FF` | `...` |
| GIF  | `47 49 46 38 39 61` または `47 49 46 38 37 61` | `GIF89a` / `GIF87a` |
| PDF  | `25 50 44 46 2D` | `%PDF-` |
| ZIP  | `50 4B 03 04` | `PK..` |
| SQLite | `53 51 4C 69 74 65 20 66 6F 72 6D 61 74 20 33 00` | `SQLite format 3.` |
| ELF (Linux実行ファイル) | `7F 45 4C 46` | `.ELF` |
| Mach-O (macOS実行ファイル, 64bit) | `CF FA ED FE` | （リトルエンディアン） |
